# Objective
## Commit and Push

---

## Description
There are many changes on the repo that have not yet been committed or pushed. The files, context, and information involved need to be identified. Commits should not be made in one shot; instead, they should be created group by group, based on their purposes and context.

---

## Primary Objective
Identify the files, context, and information for the uncommitted changes and create separate commits for them, grouped by purpose and context, rather than committing everything in one shot.

---

## Supporting Tasks

### Identify Changes
- Identify the files, context, and information for the changes on the repo

### Create Commits
- Create separate commits based on groups according to their purposes and context
- Do not commit all changes in one shot

---

## Detailed Breakdown

### Commit Grouping
- Changes must be grouped by purpose and context before committing
- Committing everything in a single commit is explicitly not allowed
