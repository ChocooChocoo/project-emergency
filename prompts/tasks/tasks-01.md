# Objective
## Create Reusable Component for Modal

**ROLE:** Frontend developer implementing a reusable modal component system.

---

## Description
The objective is to create a reusable component for Modal. The sweetalert feedback mechanism will be replaced by modals sourced from /template/modals.html. Only the Success modal and Danger modal will be used, and these will serve as the default alert or feedback going forward. A sample button will be added to the dashboard for testing purposes. The feedback contents must cover success, error, warning, and other important general feedback types.

---

## Primary Objective
Create a reusable Modal component that replaces sweetalert as the default feedback mechanism, using the Success and Danger modals from /template/modals.html.

---

## Secondary Objectives
- Add a sample button in the dashboard for testing the new modal component.
- Ensure feedback content types cover success, error, warning, and other important general feedbacks.

---

## Supporting Tasks

### Modal Replacement
- Replace sweetalert usage with modals from /template/modals.html
- Use only the Success modal and Danger modal
- Set these modals as the default alert/feedback mechanism

### Testing
- Add a sample button in the dashboard for testing

### Feedback Content
- Include success feedback
- Include error feedback
- Include warning feedback
- Add other important general feedbacks as needed

---

## Detailed Breakdown

### Modal Source and Scope
The modals must come from /template/modals.html. Only two modal types are in scope: Success modal and Danger modal. These two replace sweetalert entirely as the default alert/feedback.

### Feedback Content Types
The feedback contents should include success, error, warning, and etc. — meaning additional important general feedback types beyond these three should also be added.
